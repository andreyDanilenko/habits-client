## **Функциональные требования (FR) к модулю «Задачи и Проекты»**
### *(ClickUp-подобный с интеграцией в ERP)*

### **1. Управление жизненным циклом задачи**
*(Базовые операции)*

*   **FR-1: Создание задачи.** Система должна позволять пользователю создавать новую задачу, указывая как минимум её название.
    *   *Дополнение:* Быстрое создание (горячие клавиши `C`) из любого места системы.
    *   *Дополнение:* Создание задачи из шаблона (готовый список подзадач).
*   **FR-2: Статусы.** Пользователь должен иметь возможность изменять статус задачи.
    *   *Дополнение:* Система должна поддерживать **кастомные статусы** для разных списков/проектов (не только To Do/In Progress/Done, но и "Согласование", "Верстка", "Ревью").
    *   *Дополнение:* **Переходы (Transitions)**. Система должна позволять настроить правила: из статуса "В работе" нельзя перейти в "Готово", минуя статус "Проверка".
*   **FR-3: Удаление и архивация.** Система должна позволять удалять задачу (с перемещением в корзину) или архивировать её для скрытия из активных рабочих областей.
    *   *Дополнение:* **Восстановление.** Администратор должен иметь возможность восстановить задачу из корзины в течение 30 дней.

### **2. Атрибуты и контент задачи**
*(Информационное наполнение)*

*   **FR-4: Описание (Rich Text).** Система должна предоставлять текстовый редактор для описания задачи с поддержкой форматирования.
    *   *Дополнение:* Поддержка **Markdown** и вложенных чек-листов внутри описания.
*   **FR-5: Назначение исполнителей.** Система должна позволять назначать одного или нескольких пользователей ответственными за выполнение задачи.
*   **FR-6: Даты и время.** Пользователь должен иметь возможность установить дату начала и крайний срок выполнения задачи.
    *   *Дополнение:* **Длительность.** Автоматический расчет длительности (кол-во дней между стартом и дедлайном).
*   **FR-7: Приоритеты.** Система должна поддерживать уровни приоритетности (например: Urgent, High, Normal, Low).
    *   *Дополнение:* **Цветовая индикация** приоритетов на досках и в списках.

### **3. Организация и связи**
*(Иерархия и взаимосвязи)*

*   **FR-8: Подзадачи.** Система должна позволять создавать иерархию задач, добавляя неограниченное количество подзадач.
    *   *Дополнение:* Прогресс-бар главной задачи, который заполняется по мере выполнения подзадач.
*   **FR-9: Зависимости.** Система должна позволять связывать задачи типами «Блокирует» или «Ожидает».
    *   *Дополнение:* **Умные уведомления.** Если задача заблокирована, система предлагает уведомить владельца блокирующей задачи.
*   **FR-10: Теги.** Пользователь должен иметь возможность добавлять текстовые метки для быстрой фильтрации.
    *   *Дополнение:* Управление тегами (создание/удаление тегов в общем хранилище проекта).

### **4. Совместная работа и коммуникация**
*(Обсуждения)*

*   **FR-11: Комментарии.** Система должна позволять пользователям оставлять комментарии с возможностью упоминания (@mention) коллег.
    *   *Дополнение:* **Ответы в ветке (Threads).** Возможность отвечать на конкретный комментарий, не смешивая обсуждения.
*   **FR-12: Вложения.** Система должна поддерживать загрузку файлов и их привязку к конкретной задаче.
*   **FR-13: Наблюдатели.** Пользователь должен иметь возможность добавлять участников в качестве «наблюдателей».
    *   *Дополнение:* Автоматическое добавление в наблюдатели (автор задачи и исполнитель подписываются автоматически).

### **5. Автоматизация и кастомизация**
*(Гибкость)*

*   **FR-14: Настраиваемые поля.** Система должна позволять администратору создавать уникальные поля разных типов (число, выпадающий список, формула).
    *   *Дополнение:* **Типы полей:** Флажок (Checkbox), Ссылка (URL), Пользователь (User), Связанная задача (Task Relation).
*   **FR-15: Отслеживание времени.** Система должна включать встроенный таймер или ручной ввод времени.
    *   *Дополнение:* **Отчет по времени.** Возможность видеть, сколько времени потрачено на проект суммарно.

### **🔥 НОВЫЙ РАЗДЕЛ 6: Интеграция с ERP (Ключевое преимущество)**
*(Связь задач с деньгами, клиентами и складом)*

*   **FR-16: Привязка к объектам CRM.** Система должна позволять прикреплять задачу к любой сущности ERP (Сделка, Счет, Клиент, Проект).
    *   *Детали:* В интерфейсе задачи должен быть виджет "Связано с" с поиском по компаниям/счетам.
*   **FR-17: Виджет задач в сделке.** При просмотре карточки клиента или сделки, система должна отображать блок "Активные задачи" с возможностью создать новую задачу, не покидая карточку.
*   **FR-18: Авто-создание задач (Триггеры).** Система должна уметь создавать задачу при наступлении события в CRM.
    *   *Пример:* Если клиент не платит 3 дня → Создать задачу "Позвонить должнику" для финансового менеджера.

### **🔥 НОВЫЙ РАЗДЕЛ 7: Пользовательский интерфейс (Views & UX)**
*(Как пользователь видит задачи)*

*   **FR-19: Множество представлений (Views).** Система должна отображать одни и те же задачи в разных форматах:
    *   **Список (List):** Таблица с колонками.
    *   **Доска (Board):** Канбан с колонками-статусами.
    *   **Календарь (Calendar):** Задачи с датами на календарной сетке.
    *   **Гант (Timeline):** Лента задач с зависимостями.
*   **FR-20: Сохраненные фильтры.** Пользователь должен иметь возможность настроить сложный фильтр и сохранить его как отдельную вкладку.
    *   *Пример:* Вкладка "Мои просроченные задачи".
*   **FR-21: Глобальный поиск.** Система должна искать по названию задачи, описанию, комментариям и ID.
    *   *Детали:* Поиск с подсветкой результатов.

### **🔥 НОВЫЙ РАЗДЕЛ 8: Безопасность и Администрирование**
*(Права доступа)*

*   **FR-22: Права доступа (Permissions).** Система должна гибко настраивать, кто что может делать.
    *   *Роли:* Только автор может удалить задачу. Менеджер может менять статус у любых задач. Бухгалтер видит задачи, но не может их редактировать.
*   **FR-23: История изменений (Audit Log).** Система должна хранить лог всех изменений задачи (кто, когда и что поменял).

### **🔥 НОВЫЙ РАЗДЕЛ 9: Производительность и Масштабирование**
*(Технические требования)*

*   **FR-24: Массовые действия (Bulk Actions).** Пользователь должен иметь возможность выделить 50 задач галочками и одним кликом:
    *   Сменить исполнителя.
    *   Сменить статус.
    *   Удалить.
    *   Добавить тег.
*   **FR-25: Экспорт данных.** Система должна позволять выгружать список задач в Excel/CSV для дальнейшей отчетности.
