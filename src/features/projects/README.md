# Проекты — интеграция модулей

Проект — гибкий контейнер: объединяет сущности из разных модулей (CRM, в будущем Tasks, HR). Один проект может содержать только CRM, другой — CRM + задачи, третий — только задачи. Полная спецификация: **`docs/PROJECTS_SPEC.md`** (в корне репозитория).

## Как создать проект и наполнить его текущим CRM

1. **Откройте раздел «Проекты»** в меню (модуль должен быть включён в воркспейсе).
2. Нажмите **«Создать проект»**, введите название (и при желании описание), нажмите **«Создать»**.
3. Вы попадёте на **детальную страницу проекта**. Там три вкладки: **Контакты**, **Компании**, **Сделки**.
4. На каждой вкладке нажмите **«Добавить контакты»** / **«Добавить компании»** / **«Добавить сделки»**.
5. В модалке отобразятся сущности CRM воркспейса (уже привязанные к проекту скрыты). Отметьте нужные и нажмите **«Добавить (N)»**.
6. В проекте появятся выбранные контакты/компании/сделки. Из списка на вкладке можно перейти в карточку сущности в CRM или отвязать её от проекта (крестик).

Так вы можете один раз создать проект (например, «Клиент ООО Ромашка») и собрать в него все контакты, компании и сделки, относящиеся к этому клиенту.

## Как привязать сущность к проекту из карточки CRM

- Откройте карточку контакта/компании/сделки в CRM.
- Вкладка **«Проекты»** → **«Добавить в проект»** → выберите один или несколько проектов → **«Добавить»**.
- Ссылка **«Открыть»** у проекта ведёт на детальную страницу проекта.

## Поведение

- Одна сущность может быть в нескольких проектах.
- Удаление из проекта только отвязывает связь; контакт/компания/сделка в CRM остаётся.
- Права: кнопки добавления/удаления в проекте видны при праве `CRM_CREATE`.

## Типы сущностей (entityType в API)

- `crm_contact` — контакт
- `crm_company` — компания
- `crm_deal` — сделка

## Где используется

- **CrmContactDetailPage** — вкладка «Проекты», компонент `ProjectEntityPanel` с `entity-type="crm_contact"`.
- **CrmCompanyDetailPage** — вкладка «Проекты», `entity-type="crm_company"`.
- **CrmDealDetailPage** — вкладка «Проекты», `entity-type="crm_deal"`.

## Компоненты

- `ProjectEntityPanel` — список проектов сущности, кнопка «Добавить в проект», удаление из проекта с подтверждением.
- `ProjectAddModal` — выбор проектов (чекбоксы) и добавление выбранных.
- `useEntityProjects` — композабл: загрузка проектов сущности, список доступных для добавления, `addToProjects`, `removeFromProject`.

## API (entity/project)

- `projectService.getProjectIdsForEntity(workspaceId, entityType, entityId)` — ID проектов сущности.
- `projectService.attachEntity(workspaceId, projectId, { entityType, entityId })` — привязать к проекту.
- `projectService.detachEntity(workspaceId, projectId, entityType, entityId)` — отвязать.
